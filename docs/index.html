Mainsail-CNC

A maintained fork of Mainsail — the popular web interface for Klipper-powered 3D printers — extended with native controls for CNC mills, routers, lasers, and plasma cutters.

GitHub Mainsail Docs Architecture API Reference Installation

CNC Dashboard

Six purpose-built panels replace the traditional 3D-printing dashboard. Each panel is independently toggleable and layout-preserving across desktop, tablet, and widescreen viewports.

CNC Status

Printer state, active filename, feed override, requested feed, max velocity, host load, free RAM, and auto-loaded CAM metadata for the active file.

DRO

3-axis digital readout with machine position, work position, computed offset, axis limits, homed flags, G90/G91 mode indicator, and live velocity — all in monospace.

Jog Panel

Home All/XY/Z, XY directional pad, Z +/- buttons, step-size selector (1–100mm), configurable feedrates, disable steppers, keyboard navigation, and emergency stop (M112). Rate-limited at 50ms.

WCS Offset Preview

Interactive SVG work coordinate system manager. G54–G59 with distinct colored crosshairs, machine↔SVG coordinate mapping, hover tooltips, click-to-move (G53/G1 with 4‑decimal precision), snap-to-grid (5–100mm), stock size visualization per WCS with bounds validation and centered labels, eye‑icon per-offset toggle, legend grid, proportional SVG from machine aspect ratio. All settings persist to localStorage.

Spindle & Coolant

Spindle ON/OFF/CCW with RPM input (0–24000). Flood and mist coolant toggles. Safety confirmation dialogs gated by machine profile.

MDI Console

G-code manual data input with quick commands (G20/G21/G90/G91), WCS shortcut buttons (G54–G59), and full command history.

Key CNC Features

  • Machine coordinate system (G54–G59) with Klipper plugin
  • G10 L2/L20 work offset support via custom Klipper extra
  • WCS offsets survive Klipper restart (JSON persistence)
  • CAM metadata extraction (EstlCam, FreeCAD, Fusion 360, V-Carve)
  • CNC metadata display on G-code file cards (tool, spindle, feeds)
  • CNC Mode toggle in G-code viewer (wireframe rendering)
  • .nc file extension support in file browser
  • Moonraker CNC agent with dedicated /server/cnc/* API
  • Machine profile system (spindle, coolant, probe, safety rules)
  • Safety confirmations for spindle start and zero reset
  • Homing-before-offsets enforcement
  • MDI-renamed console (CNC terminology throughout)
  • Custom jog scripting (SAVE_GCODE_STATE/G91/G1/RESTORE)
  • 6-axis offset preview with stock visualization
  • Floating dashboard panels — drag, resize, dock with three-phase animation
  • File-type icons & colors in Config Files panel (config, python, json, yaml, script, markdown, text, log, backup, image, webcam, printer)
  • Sortable columns: click Name, Filesize, Type, or Last Modified headers
  • Hide backup files toggle — filters .bak/.bkp/.backup and files with date pattern
  • All 3D-printer features removed — bed mesh, Z-offset, filament, MMU, Spoolman, extruder, temperature presets
  • Farm page repurposed from multi-printer to multi-CNC (/allCncMachines)
  • History filters to .gcode files with CNC-relevant columns
  • Editor persists open file across reloads (?editorFile= query param)

Architecture

Frontend

Vue 3.5 + Vuetify 3 + TypeScript + Pinia

Six CNC dashboard panels, WCS SVG preview, CAM metadata viewer, and cncMode-gated layouts. Built on the complete Mainsail Vue 3 migration.

Backend — Moonraker CNC Agent

Python · moonraker-cnc-agent/

488-line Python component registering 12 endpoints under /server/cnc/* for spindle, coolant, WCS, jog, units, and settings management. Includes settings persistence and machine profile loading.

Klipper Extra — Work Coordinate Systems

Python · klipper-extras/work_coordinate_systems.py

187-line Klipper plugin implementing G10 L2/L20, six offset tables (G54–G59), G53 machine-coordinate mode, WCS_STATUS command, SAVE_WCS command, and auto-load on Klipper startup.

Klipper Macros

G-code · klipper-macros/

WCS selector macros (WCS_1–WCS_6), per-axis zeroing (ZERO_X/Y/Z/ALL), and scaffold macros for safe-Z retract and parking.

CAM Metadata Extractor

Python · scripts/cnc_metadata_extractor.py

CLI tool that parses CAM-generated G-code from EstlCam, FreeCAD, Fusion 360 MPCNC, and V-Carve, producing .cnc-meta.json sidecars with tool, spindle, feed, and operation data.

API Endpoints

GET /server/cnc/state

Combined CNC state: spindle, coolant, units, WCS, profile, capabilities, safety rules, and settings.

GET|POST /server/cnc/spindle

Query or set spindle state (off/CW/CCW) and RPM. Maps to M3/M4/M5.

GET|POST /server/cnc/coolant

Query or toggle flood and mist coolant. Maps to M7/M8/M9.

GET|POST /server/cnc/units

Query or switch between G20 (inch) and G21 (mm).

GET /server/cnc/wcs

Active WCS + per-WCS offset list for G54–G59.

POST /server/cnc/wcs/select

Switch active work coordinate system. Maps to G54–G59.

POST /server/cnc/wcs/set-zero

Set work zero on specified axes. Uses G10 L20.

POST /server/cnc/jog

Distance-based jog command with rate limiting.

GET|POST /server/cnc/settings

Read and persist CNC dashboard settings to disk.
